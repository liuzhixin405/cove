# Cove 需求说明书

文档版本：v1.0
编写日期：2026-06-15
适用版本：Cove v5.x（含 CovePhone v4.0.5）

---

## 1. 项目背景与立项动机

在 AI 编程助手快速普及的背景下，开发者对“可控、可扩展、低门槛、跨平台”的智能开发工具需求显著增长。现有方案常见问题：

1. 依赖重量级 IDE 或云端环境，部署与迁移成本高。
2. 多模型/多提供商切换复杂，配置割裂。
3. 工具调用缺乏权限边界，自动化能力与安全性难兼顾。
4. 长链路任务（多步骤改造、并发执行、会话恢复）支持不足。
5. 成本与速率限制不可见，团队难以持续使用。

基于以上痛点，Cove 的产品目标是：构建一个终端原生、单二进制、面向工程实践的 AI Coding Agent 平台，并在桌面与移动端形成一致能力体验。

---

## 2. 产品定位

Cove 是一款面向开发者与技术团队的 CLI AI 编程助手。

核心定位：

1. 终端优先：在本地开发流中直接可用，不依赖特定 IDE。
2. 工程化代理：不仅能对话，还能执行文件、命令、检索、计划编排等任务。
3. 可控自动化：通过权限模式、护栏、检查点实现“敢自动、可回退”。
4. 多模型中枢：统一接入主流模型与兼容接口，降低供应商锁定。
5. 持续进化：内建记忆、技能与自学习机制，长期提升任务完成质量。

---

## 3. 目标用户与典型场景

### 3.1 目标用户

1. 个人开发者：希望快速完成编码、重构、排障与文档工作。
2. 小型研发团队：希望在统一 CLI 工作流中共享技能与规范。
3. AI 工程团队：需要可演示、可量化、可扩展的 Agent 系统。
4. 移动优先用户：希望在 Android 端进行轻量协作与远程辅助。

### 3.2 典型场景

1. 在终端中发起单次任务（代码生成、分析、修复）。
2. 进入 REPL 进行多轮协作，动态切换模型与权限模式。
3. 执行多步骤计划（含依赖关系与并发子任务）。
4. 在长会话中查看历史、恢复上下文、压缩上下文与导出成果。
5. 对文件改动自动打检查点并支持撤销。
6. 观察成本与速率限制，避免预算失控。
7. 在移动端继续对话与配置模型。

## 4. 产品目标（可验收）

### 4.1 业务目标

1. 降低 AI 工具接入门槛：实现单二进制分发与跨平台运行。
2. 提升任务闭环率：支持从“分析-计划-执行-回退-复盘”的全链路。
3. 增强可持续使用性：成本可见、权限可控、错误可诊断。

### 4.2 能力目标

1. 支持 3 类原生提供商 + 多个 OpenAI 兼容提供商。
2. 提供 20+ REPL 命令和完整工具编排能力。
3. 支持后台任务队列、计划执行器、子智能体与团队协作。
4. 支持会话恢复、记忆管理、自学习整合。
5. 支持 Android 伴生端（CovePhone）。

## 5. 范围定义

### 5.1 本期范围（已实现范围）

1. CLI 单次调用与交互式 REPL。
2. 多提供商接入与模型切换。
3. 文件、命令、检索、网络、浏览器等 Agent 工具。

7. 记忆系统、自学习与技能系统。
8. 成本追踪、速率限制、诊断体系。

10. Android 端 CovePhone。

## 6. 功能需求清单

> 说明：以下需求按“需求描述 + 当前实现状态”组织，状态均基于现有代码与文档可验证能力。

### FR-01 交互入口与运行模式

需求：系统需支持单次命令模式与交互式 REPL 模式，满足快速执行与持续协作两类需求。
验收：支持 `-p/--print` 与默认 REPL 启动；支持 `--version`、`--help`、`--doctor`、`--config`、`--resume` 等参数。
状态：已实现。

### FR-02 多模型与多提供商接入

需求：系统需支持原生与兼容提供商，允许动态切换模型、密钥与接口地址。
验收：支持 provider/model/api-key/base-url 命令；支持环境变量与配置文件读取。
状态：已实现。

### FR-03 配置分层与覆盖策略

需求：系统需支持环境变量、用户配置、项目配置三层优先级，确保团队与个人配置可并存。
验收：配置优先级明确且可通过命令查看当前配置。
状态：已实现。

### FR-04 Agent 工具能力

需求：系统需提供读写文件、文本检索、命令执行、网页获取/搜索、浏览器渲染等工具。
验收：工具注册可见，且可在会话中被模型调用。
状态：已实现。

### FR-05 权限与安全控制

需求：系统需支持不同权限模式（如默认/计划/自动/旁路）并对高风险工具调用进行控制。
验收：可切换模式；关键写操作可受权限策略约束。
状态：已实现。

### FR-06 护栏与鲁棒性

需求：系统需具备工具循环检测、快速失败熔断、崩溃恢复等机制。
验收：出现异常调用模式时可阻断；并发执行中 panic 可被捕获。
状态：已实现。

### FR-07 计划执行器与并发编排

需求：系统需支持声明式任务计划，具备依赖关系解析与并发子任务执行能力。
验收：支持 DAG 拓扑执行、并发上限控制、计划状态可追踪。
状态：已实现。

### FR-08 后台任务与异步交互

需求：系统需支持后台任务创建、队列管理、查看与取消，避免阻塞用户输入。
验收：支持 `/tasks`、`/stop` 等命令；运行态可见。
状态：已实现。

### FR-09 子智能体与团队协作

需求：系统需支持子智能体派发、团队创建与消息传递，适配复杂协同任务。
验收：支持 `agent`、`team_create`、`send_message` 等能力。
状态：已实现。

### FR-10 定时自动化

需求：系统需支持 cron 语义的定时任务调度。
验收：可创建周期性后台任务并查看执行状态。
状态：已实现。

### FR-11 会话生命周期管理

需求：系统需支持会话历史、恢复、导出、压缩，降低长会话上下文成本。
验收：
  1. 支持 `/history`、`/resume`、`/export`、`/compact`。
  2. 针对历史会话加载具备**智能降噪机制**：自动识别并剔除无实际语义的系统单字和指令标题（如 `write`, `read`, `grep`, `cd` 等），以首句核心问题生成精确的菜单导航。
  - **渐进式分色色彩还原**：恢复历史会话时，智能展示最后 4 轮的历史对话细节，采用高反差颜色标识 User 指令与 Assistant 应答，精简 Tool 返回日志，大幅减少上下文恢复的短期记忆中断。
状态：已实现。

### FR-12 检查点与回退

需求：系统需在写操作前自动创建可回退检查点，支持用户撤销。
验收：支持自动 checkpoint、`/undo`、`/checkpoints`。
状态：已实现。

### FR-13 记忆系统与自学习

需求：系统需支持持久记忆写入、自动提取、跨会话整合（dream），提升长期任务质量。
验收：支持 `/memory`、`/dream` 与自动提取流程。
状态：已实现。

### FR-14 技能系统

需求：系统需支持内置技能与用户自定义技能，并根据上下文条件加载。
验收：支持 `skills_list`、`skill_view`，支持磁盘技能文件加载。
状态：已实现。

### FR-15 费用与配额治理

需求：系统需对 token 与费用进行实时统计，并提供预算控制与速率限制可视化。
验收：支持 `/cost`、`/budget`、`/ratelimit`。
状态：已实现。

### FR-16 诊断与可维护性

需求：系统需提供结构化诊断能力，覆盖配置、网络、模型、执行等常见问题。
验收：支持快速/完整诊断与错误码体系。
状态：已实现。

### FR-17 外部生态集成

需求：系统需支持 MCP 与插件机制，便于扩展到外部工具与企业系统。
验收：支持 `/mcp`、`/plugin` 命令与相关内部模块。
状态：已实现。

### FR-18 移动端协同

需求：系统需提供 Android 伴生应用，实现移动端聊天、配置与真实模型调用。
验收：CovePhone APK 可用，支持配置与会话。
状态：已实现。

### FR-19 符号级 AST 代码库大地图 (Repository Map)

需求：系统需支持全项目符号关系感知，在低 Token 损耗下使 Agent 具备大中型复杂工程的全局架构鸟瞰能力。
验收：
  1. 通过原生 `go/parser` 提取 Go 符号依赖图与接收器（Receiver/Methods），采用轻状态逻辑扫描 Python/TS，零外部 CGO 依赖。
  2. 实现类似 PageRank 的交叉引用计算，并对全局符号关系和出现频率进行相关度排序（最强 Top 50 输出）。
  3. 支持基于文件最新修改时间戳（$mtime$）的增量并发安全防抖缓存。
状态：已实现。

### FR-20 搜索引擎时效真实性核验 (Search Grounding)

需求：系统需结合高置信度搜索引擎服务，对 AI 动态生成的外部依赖、流行 API 方法/版本进行真实性事实核查，打击常识幻觉。
验收：
  1. 实现 `websearch` 对 `TAVILY_API_KEY`、`BRAVE_API_KEY` 环境变量的底层联动嗅探，引入并联 RAG 净化结果。
  2. 提供 DuckDuckGo 刮削兜底机制，保持高可靠的免 SEO 偏见联网获取能力。
状态：已实现。

## 7. 非功能需求

### NFR-01 可移植性

1. 支持 Windows / macOS / Linux。
2. 单二进制分发，降低部署复杂度。

### NFR-02 性能与响应

1. 支持流式输出。
2. 支持异步任务，避免 REPL 输入阻塞。

### NFR-03 稳定性

1. 异常情况下不中断主流程（如并发 panic recover）。
2. 关键路径具备测试覆盖与诊断能力。

### NFR-04 安全与可控

1. 提供权限模式与护栏策略。
2. 写操作前检查点，降低误操作损失。

### NFR-05 可观测性

1. 成本、速率限制、任务状态、诊断结果可查询。
2. 支持调试模式与结构化错误码。

## 8. 展示建议指标

1. 任务闭环率：从需求输入到可运行输出的完成比例。
2. 平均交付时长：典型任务（修复 bug、新增命令、重构模块）完成时间。
3. 并发效率：计划执行器并发后相对串行耗时下降比例。
4. 安全可控性：高风险操作确认率、回退成功率。
5. 经济性：单任务平均 token 成本与预算命中率。
6. 稳定性：异常恢复成功率、长会话恢复成功率。
