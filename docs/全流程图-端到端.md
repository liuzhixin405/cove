# Cove 全流程图（总览 + 模块内细化）

说明：本文件在端到端总流程基础上，补充各大模块内部运作流程，便于做技术展示与系统讲解。

## 1) 端到端总流程（保留总览）

```mermaid
flowchart TD
    A[启动 cove 进程] --> B[解析 CLI 参数]
    B --> C{参数分支}

    C -->|--version/--help/--doctor/--config| C1[直接执行并退出]
    C -->|正常启动| D[bootstrapApp 初始化]

    D --> D1[配置加载与迁移]
    D --> D2[构建 Provider/Engine/Registry]
    D --> D3[初始化 Permission/MCP/Skills/Memory/Plugin]
    D --> D4[启动诊断 + Banner]

    D4 --> E{运行模式}
    E -->|--dump-system-prompt| E1[打印系统提示词后退出]
    E -->|-p/--print| F[一次性模式 runPrintMode]
    E -->|默认| G[交互模式 runREPL]

    F --> F1[构造消息和附件]
    F1 --> H[runChatInteractionMessage]
    H --> I[Engine.RunWithStream]
    I --> J[输出结果]

    G --> G1[REPL 读入 + 自动补全]
    G1 --> G2{输入类型}
    G2 -->|斜杠命令| G3[命令注册表执行]
    G2 -->|普通消息| G4[任务入队 replTaskRunner]
    G4 --> H
    G3 --> G1

    J --> Z[进程结束]
    G1 -->|/exit 或 EOF| Z
```

## 3) REPL 主循环与异步任务队列

```mermaid
flowchart TD
    A[runREPL 启动] --> B[构建命令列表和补全器]
    B --> C[创建 replTaskRunner]
    C --> D[读取输入]

    D --> E{终端事件}
    E -->|Ctrl+C 且任务运行中| E1[CancelRunning]
    E -->|EOF 或 /exit| E2[SaveSession + 退出]
    E -->|普通输入| F{是否权限提示输入通道}

    F -->|是| F1[转发到 PermissionPrompt]
    F -->|否| G{输入类型}

    G -->|/xxx 命令| H[命令执行]
    G -->|continue/继续| I[重试失败任务或恢复草稿]
    G -->|普通消息| J[Enqueue 到任务队列]

    J --> K{可合并排队任务?}
    K -->|是| K1[合并补充要求]
    K -->|否| K2[追加队列]

    K1 --> L[startNextLocked]
    K2 --> L
    L --> M[后台 goroutine run]
    M --> N[runChatInteractionMessage]
    N --> O{执行结果}
    O -->|成功| O1[清理中断草稿]
    O -->|失败| O2[保存 pendingFailedMsg + interrupted draft]
    O1 --> P[finishLocked + 尝试下一个任务]
    O2 --> P
    P --> D
    H --> D
    I --> D
```

## 4) 交互层流程（runChatInteractionMessage）

```mermaid
flowchart LR
    A[开始一次交互] --> B[初始化 Spinner 与输出缓冲]
    B --> C[最多 3 次自动重试]
    C --> D[调用 Engine.RunWithStream / RunMessageWithStream]

    D --> E{收到 delta/reasoning}
    E -->|是| E1[停止 Spinner + 流式打印]
    E -->|否| F

    D --> F{是否报错}
    F -->|否| G[返回成功]
    F -->|是| H{是否瞬时错误且可重试}
    H -->|是| C
    H -->|否| I[输出错误并返回]
```

## 5) 引擎回合主流程（Engine.RunWithStream）

```mermaid
flowchart TD
    A[接收用户消息] --> B[写入 messages]
    B --> C[构建 SystemPrompt: tools/skills/memory/context]
    C --> D[for iter < MaxIterations]

    D --> E[Provider.Chat]
    E --> F{响应类型}
    F -->|纯文本| G[onDelta 流式输出]
    F -->|带 tool calls| H[解析工具调用]

    H --> I[并发安全工具并行(上限8) + 非并发工具串行]
    I --> J[executeTool]
    J --> K[将结果回灌为 tool message]
    K --> D

    G --> L[saveSession + cost/ratelimit 更新]
    L --> M[runTurnEndPipeline]
    M --> N[返回最终文本]

    D --> O{token 超阈值且可压缩?}
    O -->|是| O1[Compact 历史]
    O -->|否| E
    O1 --> E
```

## 6) 单次工具执行内部流程（executeTool）

```mermaid
flowchart TD
    A[按名称查找 Tool] --> B{存在?}
    B -->|否| B1[返回 unknown tool]
    B -->|是| C[Guardrail.BeforeCall]

    C --> D{Guardrail 决策}
    D -->|Block| D1[返回阻断错误]
    D -->|Warn/Allow| E[构造 tool.Context]

    E --> F{bash 分类器检查}
    F -->|危险命令| F1[直接拒绝]
    F -->|通过| G[Validate 输入]

    G --> H[Tool.CheckPermissions]
    H --> I[PermissionManager.Check]
    I --> J{allow/ask/deny/bypass}

    J -->|deny| J1[返回拒绝]
    J -->|ask| J2[触发 PermissionPrompt y/n/a]
    J2 -->|拒绝| J3[返回拒绝]
    J2 -->|允许| K[继续执行]
    J -->|allow/bypass| K

    K --> L{write/edit?}
    L -->|是| L1[异步自动 checkpoint]
    L -->|否| M
    L1 --> M[调用 t.Call]

    M --> N{执行错误?}
    N -->|是且瞬时错误| N1[重试一次]
    N -->|是且仍失败| N2[Guardrail.AfterCall(error) + 返回错误]
    N -->|否| O[Guardrail.AfterCall(success)]

    N1 --> O
    O --> P[trackFileChanges + 输出裁剪 + 条件技能注入 + 子目录提示]
    P --> Q[返回结果给模型]
```

## 7) 权限系统内部决策流程（permission.Manager.Check）

```mermaid
flowchart TD
    A[输入: toolName + toolInput + defaultDecision] --> B[匹配 deny 规则]
    B --> C{命中?}
    C -->|是| C1[DDeny: denied by policy rule]
    C -->|否| D{mode == bypass 且可用?}

    D -->|是| D1[DBypass]
    D -->|否| E[匹配 allow 规则]
    E --> F{命中?}
    F -->|是| F1[DAllow]
    F -->|否| G[匹配 ask 规则]
    G --> H{命中?}
    H -->|是| H1[DAsk]
    H -->|否| I{mode}

    I -->|auto| I1[默认安全则 allow 否则按默认决策]
    I -->|plan| I2[仅允许只读; 非只读转 deny]
    I -->|default| I3[返回 defaultDecision]
```

## 8) 计划执行器内部流程（execute_plan / plan executor）

```mermaid
flowchart TD
    A[todowrite 产生任务清单] --> B[execute_plan 调用 PlanExecuteFunc]
    B --> C[FromRuntime 构建 Plan]
    C --> D[topologicalSort 依赖层级]
    D --> E{存在环?}
    E -->|是| E1[所有任务 failed: circular dependency]
    E -->|否| F[逐层执行]

    F --> G{parallel 且同层>1?}
    G -->|是| G1[并发执行, sem 限制 MaxParallelAgents=4]
    G -->|否| G2[串行执行]

    G1 --> H[runTask]
    G2 --> H

    H --> H1[读取依赖输出 + 注入消息]
    H1 --> H2[delegator.Delegate 调子智能体]
    H2 --> H3{成功?}
    H3 -->|否| H4[按 maxRetries 重试 默认1]
    H3 -->|是| H5[标记 done + 回写 runtime]
    H4 --> H6{重试后仍失败?}
    H6 -->|是| H7[标记 failed]
    H6 -->|否| H5

    H5 --> I[更新 completed]
    H7 --> I
    I --> J{有依赖失败?}
    J -->|是| J1[下游 pending 标记 skipped]
    J -->|否| K
    J1 --> K[进入下一层]
    K --> L[汇总 ExecutionResult]
```

## 10) 会话、压缩、检查点流程

```mermaid
flowchart TD
    A[每轮对话后 saveSession] --> B[更新消息、token、cost、title]
    B --> C[写入 session store]

    C --> D{token > 阈值且轮次满足?}
    D -->|是| E[Compact]
    D -->|否| F[继续]

    E --> E1[先 trimOldToolResults]
    E1 --> E2{仍超阈值?}
    E2 -->|是| E3[调用模型总结中段历史]
    E2 -->|否| F
    E3 --> E4[重建消息: 首消息 + 摘要 + 最近尾部]
    E4 --> F

    F --> G{工具是 write/edit?}
    G -->|是| H[cpMgr.Create auto checkpoint]
    G -->|否| I[本轮结束]
    H --> I
```

- 计划执行器：internal/plan/executor.go
